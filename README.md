## ◆ 1. 这是什么？

弥不是"一个 AI 陪你聊天"。它是一个**运行在本地的虚拟 AI 软件公司**。

你扮演**董事长**，提出想法、审批关键方案。然后，一个完整的 AI 团队自动运转：

| 🎭 角色 | 📋 职责 | 🏢 比喻 |
|---------|---------|---------|
| 策略产品经理 | 理解你的想法，生成版本方案 | CEO |
| 产品经理 | 将版本拆分为大阶段 | CTO |
| 域负责人 | 将大阶段拆为中阶段，分派子任务 | 技术主管 |
| 开发工程师 | 执行具体子任务，操控 Claude Code 写代码 | 程序员 |
| 测试工程师 | 检查代码质量，确保测试通过 | QA |
| 宪法维护员 | 自动更新项目知识文档 | 技术文档 |

> **全部由 AI 驱动。你只需要对话和审批。**

## ◆ 2. 和你用过的东西有什么不同

| ❌ 你熟悉的 | ✅ 弥的做法 |

| Cursor / Copilot 的行级补全 | 项目级自动执行——从方案到代码一气呵成 |
| Claude Code 单次对话 | 多角色 AI 团队协作，决策-执行分离 |
| 手动 `git commit` | 自动中阶段 tag + 子任务 tag，两级回退 |
| 靠人记住项目规范 | 滚动**项目宪法**，AI 自维护、全局共享 |
| 被大模型绑定 | **精准上下文注入 + 原子化任务 + 滚动宪法** → 任何模型都能高质量交付，随时可换 |
| 和多个 AI 角色碎片化沟通 | 只和 CEO 对话，它指挥整个团队 |

## ◆ 3. 关键特性

### 🧩 3.1 决策-执行分离流水线

策略产品经理用便宜的模型做决策——生成 prompt、拆任务、评估质量。执行层操控 Claude Code 写代码。**决策是便宜的，执行是精准的。**

> 不是靠强模型带弱模型的分层策略。而是通过精准上下文注入、原子化任务切割和滚动项目宪法，让**任何模型**都能经济地完成高质量项目交付。

流水线支持：`暂停` → `恢复` → `自动重试` → `测试检查` → `宪法更新` → `Git 存档`

> **当前执行层基于 Claude Code 验证核心工作流。** 但 Claude Code 只能串行——子任务逐个排队执行，每个子任务都要重新注入完整上下文，token 消耗层层叠加。这是我们的过渡方案。自研 Rust API 直连引擎已在路线图上：第一个子任务建立基础后，后续子任务**并行注入代码**，共享一次上下文，速度提升数十倍，token 消耗大幅下降。

### 📜 3.2 项目宪法（滚动知识库）

每个项目自动维护一份 `CONSTITUTION.md`：

| 部分 | 内容 | 维护方式 |
|------|------|---------|
| **第 1 部分** | 用户确认的核心原则 | 不可变 |
| **第 2 部分** | 项目架构 · 文件结构 · 变更记录 | AI 自动维护 |

三层防御机制：

```
🛡️ 校验    → AI 更新后检查合法性
🛡️ 降级    → 校验失败 → 机械兜底更新
🛡️ 剪枝    → 内容过长 → 自动压缩，保留关键信息
```

> **AI 团队不再依赖你的个人记忆力。**

### 🔖 3.3 细粒度 Git 回退

```
metheus/v0.1.1                    ← 中阶段 tag
metheus/auto/v0.1.1/task-0        ← 子任务 tag
```

回退时自动 `stash` 未提交变更。你可以回到**任何一个子任务执行前的状态**。

### 🖥 3.4 任务控制台

| 标签页 | 功能 |
|--------|------|
| **执行日志** | 流水线实时日志 |
| **代码变更** | 执行前后 Diff 对比 |
| **文件树** | 项目结构一览 |
| **宪法** | 随时查看 AI 维护的项目知识 |

### 🔓 3.5 模型自由

决策层和执行层独立配置模型。随时把决策模型换成更便宜的，把执行模型换成更强的。

> **不被任何 AI 厂商绑定。**

### ❄️ 3.6 Nix 环境集成

支持 Nix Flakes。项目声明式定义开发环境，AI 团队在可复现的环境中工作。

## ◆ 4. 快速开始

### 你需要什么

| 依赖 | 说明 |
|------|------|
| Rust (stable) | 后端编译 |
| Node.js 20+ | 前端运行 |
| Tauri CLI | `cargo install tauri-cli` |
| Claude Code | 执行层代码生成 |
| Nix（可选） | 启用 Flakes 环境 |

### 三步启动

```bash
git clone https://github.com/entzauberung/metheus.git
cd metheus
npm install && npm run tauri dev
```

### 首次使用的路径

```
1. 打开弥，输入项目名称
       ↓
2. 和策略产品经理对话，描述你的想法
       ↓
3. 审批生成的版本方案
       ↓
4. 弥自动拆任务 → 写代码 → 跑测试 → 存档
       ↓
5. 在任务控制台查看进度和变更
```

## ◆ 7. 项目状态

### 为什么暂停

这个项目由我一个人全职开发。无外部资金，无团队。

前端架构刚刚从单一聊天界面升级为任务控制台——执行树、代码 Diff、四标签页。这是「弥」从"能用"走向"好用"的关键蜕变，但也带来了大量的前后端重新对齐工作。

> **我用自己吃饭的钱充 API。** 在最后一个 token 包用完之后，不得不等待下一周的生活费才能继续。

这不是项目的终结。这是弹尽粮绝时的战术性暂停。

### 当前主要技术问题

```
⚠️ 流水线状态不持久化      → 关闭后再打开，正在执行的流水线丢失
⚠️ 执行中断边界处理不够健壮  → AI prompt 失败或 Git 存档失败时直接终止
⚠️ 前端运行时状态不持久化    → 视图模式、选中状态等关闭后丢失
⚠️ 流式输出未实现           → AI 回复需等待完整响应
```

> 以上问题都是**"没做完"，不是"做错了"**。宪法系统、Git 回退、决策-执行分离流水线——这些产品级基础设施已经工作。

### 为什么仍然相信

```
✓ 多角色 AI 团队可以自动化地从"想法"走到"代码"
✓ 项目宪法让 AI 团队的产出质量不随对话轮次衰减
✓ 细粒度 Git 回退给了用户真正的安全感
✓ 便宜的决策 + 精准的执行 = 独立开发者负担得起的价格
✓ Claude Code 串行验证已通过 → 自研并行引擎可将速度提升数十倍，token 消耗大幅下降
```

每解决一个问题，就离**"用一杯奶茶钱改变世界"**更近一步。
